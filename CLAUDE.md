## Build and Test Commands

The project uses a standard Go toolchain and `make` for convenience.

### Core Development

```bash
# Run all tests with verbose output and coverage
make test
go test -v -cover ./...

# Run tests for a specific package
go test ./signer
go test ./parser

# Build (as a library, this just verifies compilation)
go build ./...
````

### Code Quality

```bash
# Format all Go code
make fmt
go fmt ./...

# Run linters (requires golangci-lint)
make lint
golangci-lint run
```

## Architecture Overview

`go-cms` is a Go library for creating and parsing CMS/PKCS\#7 messages as defined in RFC 5652. Its primary focus is on enabling the use of modern cryptographic algorithms, specifically Ed25519, within this established standard.

### Core Components

The library is organized into packages with clear responsibilities:

- **`signer/`**: Contains the logic for creating CMS `SignedData` structures. This is the core implementation for generating signatures.
- **`parser/`**: Contains the logic for parsing and verifying existing CMS messages.
- **`types/`**: Defines the core ASN.1 structures that map to the CMS specification, used by both the signer and parser.
- **`protocol/`**: Holds constants and definitions for Object Identifiers (OIDs) and other protocol-level values, such as those from RFC 8410 for Ed25519.

### Key Design Patterns

1. **Standards-Compliant**: The implementation adheres strictly to the relevant RFCs (5652 for CMS, 8410 for Ed25519 OIDs) to ensure interoperability.
2. **Idiomatic Go API**: The library exposes a clean, simple API that feels natural to Go developers, hiding the complexity of ASN.1 encoding and cryptographic operations.
3. **Zero Dependencies**: The core library has no external dependencies beyond the Go standard library and the `golang.org/x/crypto` packages.

### Testing Strategy

The project maintains a very high test coverage ratio (\~4000 lines of tests for \~1500 lines of code). The key testing strategies are:

- **Round-Trip Tests**: Ensure a message signed by the library can be successfully parsed and verified by the library.
- **Interoperability Tests**: A critical part of the test suite verifies that signatures generated by `go-cms` can be successfully verified by OpenSSL. These tests are located in `signer/openssl_verify_test.go`.
- **Test Vectors**: Uses known-good test vectors where available to ensure correctness.

## Implementation Notes

### Ed25519 Support

This library is the first open-source Go implementation to provide support for creating CMS messages signed with the Ed25519 algorithm.

- The implementation correctly uses the `id-Ed25519` Object Identifier (OID) `1.3.101.112`.
- The `SignerInfo` structure is populated according to the standards for modern elliptic curve algorithms.

## Current State

- The API is considered stable and is tracking towards a `v1.0.0` release.
- The library is well-tested but has not yet undergone a formal third-party security audit. Use in production systems should be done after careful risk assessment.
