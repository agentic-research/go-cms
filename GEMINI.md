# GEMINI.md

## Project Overview

This project is a Go library named `go-cms` that provides functionality for creating and verifying Cryptographic Message Syntax (CMS) messages, also known as PKCS#7. The key feature of this library is its support for Ed25519 signatures, which is not available in other popular Go CMS libraries. The library aims to be standards-compliant and compatible with OpenSSL. It is self-contained, with no external dependencies other than the Go standard library and `golang.org/x/crypto`.

The core logic is implemented in the `pkg/cms` directory, with `signer.go` handling signature creation and `verifier.go` handling signature verification. The project also includes a command-line tool, `cms-test-tool`, for testing purposes.

## Building and Running

The project uses a `Makefile` to streamline the build, test, and linting processes.

### Key Commands:

*   **Build the test tool:**
    ```bash
    make build
    ```
    This command compiles the `cms-test-tool` and places the binary in the `bin/` directory.

*   **Run tests:**
    ```bash
    make test
    ```
    This command runs all the unit tests for the library, including race detection and code coverage.

*   **Run OpenSSL interoperability tests:**
    ```bash
    make docker-test
    ```
    This command runs tests in a Docker container to ensure that signatures generated by `go-cms` can be successfully verified by OpenSSL.

*   **Format code:**
    ```bash
    make fmt
    ```
    This command formats the Go source code according to the standard Go style.

*   **Lint code:**
    ```bash
    make lint
    ```
    This command runs `golangci-lint` to check for style violations and potential bugs.

## Development Conventions

*   **Coding Style:** The project follows standard Go coding conventions and formatting, enforced by `gofmt`.
*   **Testing:** The project has a strong emphasis on testing, with a high test-to-code ratio. Tests include unit tests, round-trip tests, and interoperability tests with OpenSSL. New contributions are expected to include appropriate tests.
*   **Dependencies:** The project has no external dependencies, relying only on the Go standard library and `golang.org/x/crypto`.
*   **Linting:** The project uses `golangci-lint` to maintain code quality.
